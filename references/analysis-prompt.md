# Analysis Prompt Reference

分析转录文本时使用的标准提示词模板。先执行 Stage 1，再用 Stage 2 格式化输出供用户审核。

---

## Stage 1：候选片段选取

```
你是一位专业的短视频剪辑编辑。以下是一段访谈/播客的英文字幕（JSON格式）。

请分析全文，找出适合剪辑成独立短视频的片段。

选取标准：
- 可以独立理解，不依赖大量前文
- 具备以下至少一种特质：有观点、反常识、有激励性、情感鲜明、适合引用
- 时长在 20 秒到 3 分钟之间
- 开头 3 秒内有强钩子
- 以完整思想结尾，不截断句子

排除标准：
- 暖场、介绍、闲聊
- 赞助商推广
- 需要大量前文才能理解的片段
- 主要依赖视频画面的片段

对于 1 小时左右的视频，请提供 10 到 15 个候选片段。宁可多选，不要漏掉精彩内容。

对每个候选片段，提供：
- id（两位数字）
- start / end（SRT格式时间戳）
- start_seconds / end_seconds（浮点秒数）
- duration_seconds
- working_title（工作标题）
- summary（恰好两句：第一句说内容，第二句说价值）
- quality_signals（数组）

以 JSON 格式输出，结构遵循 references/clip-schema.md。

字幕内容：
{TRANSCRIPT_JSON}
```

## Stage 2：用户审核格式

```
请将 selected_clips.json 中的候选片段，格式化为以下用户友好的审核格式：

---
**[01] 为什么大多数人永远不会成功**
⏱ 03:12 → 05:47（2分35秒）

嘉宾指出大多数人失败不是因为能力不足，而是因为从未真正承诺过一件事。他用亲身经历说明，专注力才是区分成败的关键变量。

---

每个片段单独一块，用分隔线隔开。
结尾询问用户："请输入你想导出的 clip id（如 01 03 07），或输入"全部"自动选择最强片段。"
```

## Stage 3：中文字幕翻译

```
请将以下英文 SRT 字幕翻译成简体中文。

规则：
- 保留所有时间戳，不修改
- 翻译成自然的口语中文，不要逐字直译
- 保留专有名词、数字、产品名称、引用短语
- 如果一句英文跨多个字幕条目，可在相邻条目间调整文字分布，但不合并时间戳
- 不要大幅压缩字幕数量（默认失败模式是字幕太少，不是太多）
- 去除重复的自动生成片段，但保留大部分非重复条目

英文 SRT 内容：
{SRT_CONTENT}
```

## Stage 4：标题与描述生成

```
请为以下短视频片段生成标题和描述。

片段信息：
- 发言人：{SPEAKER}
- 节目/访谈：{SHOW_NAME}
- 话题：{TOPIC}
- 核心内容：{SUMMARY}

标题要求：
- 适合短视频平台传播（可反问、可反常识、有张力）
- 忠实原意，不歪曲发言人立场
- 屏幕叠加版本约 12 个中文字

描述要求：
- 140 中文字以内
- 包含：发言人、节目名称、话题、核心观点或结论

请分别输出：
1. 完整标题（可略长）
2. 屏幕叠加标题（约 12 字）
3. 描述文字
```
